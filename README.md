# QA Automation Project: SauceDemo UI Tests

Репозиторий содержит автотесты на Selenium + Java + JUnit для учебного приложения [SauceDemo](https://saucedemo.com).

## 📋 Чек-лист проверок (Test Cases)

Ниже представлен перечень автоматизированных и планируемых проверок.

### 1. Авторизация (Login Page)
- [x] **Успешный вход**: вход с валидными данными (`standard_user` / `secret_sauce`).
- [x] **Пустой пароль**: ошибка `Epic sadface: Password is required`.
- [x] **Пустой логин**: ошибка `Epic sadface: Username is required`.
- [x] **Невалидные данные**: ошибка `Epic sadface: Username and password do not match any user in this service`.

### 2. Страница товаров (Products Page)
- [x] **Отображение контента**: после логина доступен заголовок "Products".
- [x] **Добавление в корзину**:
    - [x] Добавление одного товара. Проверка его наличия в корзине.
    - [x] Добавление всех товаров. Проверка наличия всех в корзине.
    - [x] Нажатие "Add to cart" меняет текст кнопки на "Remove".
    - [ ] Нажатие "Add to cart" увеличивает счетчик на иконке корзины.
- [x] **Удаление товара**: нажатие "Remove" меняет текст обратно на "Add to cart" и уменьшает счетчик. В корзине отсутствует товар.
- [ ] **Сортировка**: проверка фильтров (A-Z, Z-A, Price Low-High, Price High-Low).

### 3. Корзина (Cart Page)
- [x] **Отображение контента**: после логина доступен заголовок "Your Cart".
- [x] **Соответствие данных**: название и цена товара в корзине совпадают с выбранными в каталоге.
- [x] **Удаление из корзины**: после удаления товар исчезает из списка.
- [ ] **Пустая корзина**: проверка состояния, когда все товары удалены (счетчик 0).

### 4. Оформление заказа (Checkout)
- [ ] **Отображение контента**: после логина доступен заголовок "Checkout: Your Information".
- [ ] **Валидация полей**: проверка обязательности полей First Name, Last Name, Zip Code.
- [ ] **Завершение заказа**: переход на финальную страницу "Checkout: Complete!" после оплаты.
- [ ] **Суммирование цен**: проверка, что Item Total + Tax = Total.---
